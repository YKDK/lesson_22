Основи синтаксису

  1. варіант без помилок
  2. варіант майжеправильно: не хватає ; після consol.log('JS') так, як при збірці коли всестане в рядок в консоль виведе помилку
  3. варіант не правильно, нема крапки з комою між виразами
  4. варіант не правильно, не хватає закриваючої лапки в першому виразі

Змінні

  1. варіант не вірно: після 'Вася' треба поставити крапку з комою, або записати змінну age з нового рядка
  2. не критична помилка - не використовується стиль lowerCamelCase
  3. варіант правильний, але якщо величина блоку змінюється то варто її записати через let
  4. варіант - правильно
  5. варіант правильний якщо для об'яви змінної викоритсовувати VAR, і викличе помилку якщо для об'яви змінної використовувати let

Типи данних

  1. варіант не вірний, значення треба помістити в зворотні лапки та добавити фігурні дужки і символ доллар змінній userAge: `фрілансер ${(userAge)}`
  2. варіант - поверне значення infinity
  3. варіант - не вірно, повернеться значення udefined
  4. варіант - вірно: ділення рядка з числовим значенням на рядок з числовим значенням повертає тип данних Number
